Теоретическая часть: Очередь с приоритетом на основе кучи
Очередь с приоритетом - это структура данных, в которой каждый элемент имеет приоритет. Элементы с более высоким приоритетом обрабатываются раньше элементов с более низким приоритетом. Очередь с приоритетом можно реализовать с помощью различных структур данных, но наиболее часто используется куча (heap), так как она обеспечивает эффективные операции вставки и извлечения минимального или максимального элемента.

Куча - это специализированное бинарное дерево, которое удовлетворяет свойству кучи:

Минимальная куча (min-heap): Значение в каждом узле не больше значений его потомков.
Максимальная куча (max-heap): Значение в каждом узле не меньше значений его потомков.
Основные операции с очередью с приоритетом на основе кучи:

Вставка (insert): Добавление нового элемента в кучу. Элемент добавляется в конец дерева, а затем "всплывает" вверх по дереву, пока не найдет правильное место.
Извлечение минимума/максимума (extract-min/extract-max): Удаление корневого элемента (минимального или максимального), затем замена его последним элементом дерева и "погружение" этого элемента вниз по дереву, пока куча не станет валидной.
Построение кучи (build-heap): Преобразование произвольного массива в кучу путем вызова операции "погружения" для каждого элемента массива.

Объяснение кода
Класс MinHeap: Определяет методы для работы с минимальной кучей.
Метод insert: Вставляет элемент в кучу и вызывает _heapify_up, чтобы поддерживать свойство кучи.
Метод extract_min: Удаляет и возвращает минимальный элемент (корень кучи), затем вызывает _heapify_down, чтобы поддерживать свойство кучи.
Метод _heapify_up: "Всплывает" элемент вверх по дереву, пока не будет восстановлено свойство кучи.
Метод _heapify_down: "Погружает" элемент вниз по дереву, пока не будет восстановлено свойство кучи.
Метод get_min: Возвращает минимальный элемент (корень кучи) без его удаления.
